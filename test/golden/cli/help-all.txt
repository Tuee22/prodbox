## prodbox rke2 delete
prodbox rke2 delete - Delete RKE2

Delete the local cluster. Default mode opens with a per-run-residue check and refuses if `aws-eks`, `aws-eks-subzone`, or `aws-test` still have live resources. `--cascade` orchestrates the full teardown (K8s drain + per-run Pulumi destroys + cluster uninstall + postflight tag sweep) as one atomic operator action; the K8s drain phase skips gracefully when no cluster is reachable. `--allow-pulumi-residue` bypasses the residue check for operator-acknowledged recovery scenarios. `--cascade` and `--allow-pulumi-residue` are mutually exclusive at parse time.
Options:
- --yes, -y - Confirm full RKE2 cluster deletion
- --cascade - Orchestrate the full teardown (K8s drain + per-run Pulumi destroys + cluster uninstall + postflight tag sweep) as one atomic operator action
- --allow-pulumi-residue - Bypass the per-run Pulumi residue refuse-path (recovery only)
- --dry-run - Render the delete plan without mutating state
- --plan-file <PATH> - Write the rendered plan to a file

Examples:
- prodbox rke2 delete --yes  # Delete the local cluster (refuses if per-run Pulumi stacks are live).
- prodbox rke2 delete --yes --cascade  # Orchestrate the full teardown as one atomic operator action.

- --dry-run - Render the revoke plan without mutating state
- --plan-file <PATH> - Write the rendered plan to a file

Examples:
- prodbox users revoke operator@example.invalid  # Disable the operator account.
- prodbox users revoke operator@example.invalid --delete  # Fully delete the operator account.

Examples:
